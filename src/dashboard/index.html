Getting Started

1. Start the server: npx tsx src/index.ts

2. Install hooks: npx tsx src/index.ts install

3. Run claude in any terminal — agents appear automatically

Panels

Agents — Click an agent to filter timeline and send instructions. Shows live tool activity, diff stats (+/-), files touched, and session duration.

Missions — Press n to create tasks. Assign to agents, set priority, add dependencies. Supports subtask progress tracking.

Usage — Real API costs from Claude Code logs. Switch periods (24H/7D/30D/All). Shows token breakdown, cost by model, context window health for active sessions, and cache hit rate.

Timeline — Click any event row to expand full tool input/output. Filter by agent by clicking an agent name.

Alerts

STUCK — No activity for 2+ min. Check the agent's terminal.

LOOP — Repeating the same tool calls. Agent may need redirection.

SPIRAL — Editing the same file repeatedly. Send a clarifying instruction.

ERRORS — 5+ failures. Review the timeline for error details.

MARATHON — Running 30+ min. Consider compacting context.

Enable browser notifications to get alerts even when this tab is in the background.

Agent Actions

• Hover an agent to see quick-action buttons: Copy Path (working directory) and Copy ID (session ID).

• Click an agent to select it, filter the timeline, and enable the instruction input.

• Use the copied path to find the right terminal window (e.g. Cmd+Tab and match the folder name).

Mission Actions

• Click a mission to expand it and see action buttons.

• Queued: Assign to an agent, start, or delete.

• Active: Mark as complete or failed.

• Blocked: Force unblock to move back to queued.

• Completed/Failed: Requeue to run again.

Tips

• Set MC_COST_PER_TOOL_CALL=0.01 to adjust estimated costs when JSONL logs aren't available.

• Data persists in ~/.claude-mission-control/data.db — survives restarts.

• Real costs are read from ~/.claude/projects/ JSONL session logs.

• The secret scanner checks every tool output for leaked API keys, tokens, and private keys.
